2.8 Creating an application window using sizers

Sizers are a great way to ensure that your GUI layout is nice and clean. They come in especially handy when you do not know exactly how much space a control needs and/or should be allowed to use, this can be the case when you internationalize your application (I18N) or for such controls as lists or grids where you like to give as much space as possible to them (or maybe as little as practical).

Please note that the following will just explain how to use sizers in Boa (note that this assumes version 0.5.x of Boa). For more detailed information about sizers you should check the wxPython documentation and you might also find the following links interesting.

http://wiki.wxpython.org/index.cgi/UsingSizers

http://wiki.wxpython.org/index.cgi/LearnSizers1

http://wiki.wxpython.org/index.cgi/wxDesigner_20Sizer_20Tutorial


We will use a wx.Frame and create a screen for address information entry. In this first tutorial we will just use one sizer to control the layout of the screen, there will be another tutorial (hopefully soon) with a layout a bit more complex using multiple sizers.

  • Close all the source files in your editor, so not to add this to the application you created previously.

  • On the palette, select the 'New' pane. Select the 'wx.Frame' button. This will create a new source file *(Frame1)*.

  • Click on the Save button (or menu File/Save) and save it as AddressEntry.py.

  • Select from the menu Edit the option Add module runner. This will add some code to your file so you can run without having to have a separate wx.App file.

  • Save the file and you can run this application, you will see just Frame1 in the title bar and a grey background.

  • Select the AddressEntry pane. Start the Designer by clicking on the button .

  • On the palette, select the 'Containers/Layout' pane. Click on the wx.Panel button to select it and click anywhere within the AddressEntry frame. This will drop the panel onto your frame.

  • On the same palette pane click on the wx.FlexGridSizer button to select it and click anywhere on the wx.Panel you just added to your frame. You should see a yellow line around your panel.

  • In the Inspector change the 'Cols' setting from '0' to '2' and the 'Rows' setting from '1' to '0', as we will have to columns of controls/widgets.

  • Post the changes, save the file and open the Designer again. In the Inspector change the Name from 'Frame1' to 'AddressEntry' and the Title from 'Frame1' to 'Address entry form'. You will see that the wx.Panel is using up all available space and if you run the application and resize the window you see that it will resize accordingly.

  • We will now add the controls, to have less work later you should try to add them in the sequence you want to have them.

    • We will have a wx.StaticText and a wx.TextCtrl for each of the fields we need. The fields are: First name, Last name, address, postal, city, country and notes.

    • To add these controls/widgets select the palette 'Basic Controls' and click wx.StaticText and click anywhere on the designer window. This will create the control and add it to the sizer. Now bring the designer window to the front again (i.e. click on ), click on the wx.TextCtrl on the palette pane and click anywhere on the designer window. This has created the control and added it to the sizer.

    • Now repeat the above for all the fields needed.

    • At this point your designer windows should look like the following image.





  • Select the Sizers pane in the Editor window and double click on flexGridSizer1 item (if you use many sizers you should probably also rename this to something a bit more meaning full).

  • Please note that the flags etc being used in the following are explained on the links mentioned above or in the wxPython documentation.

  • Now click on the first item and in the Inspector change the Border setting from '0' to '4' and the Flag setting from '0' to
    'wx.ALIGN_CENTER_VERTICAL | wx.ALL' and repeat this for each wx.StaticText control.

  • For all the wx.TextCtrl change the Border setting from '0' to '4' and the Flag setting from '0' to 'wx.ALL', with the exception of the last one which you set to 'wx.ALL | wx.EXPAND' and for this one you set the Proportion setting from '0' to '1'.

  • Select the Sizers pane again and click on the flexGridSizer1 item and select the Props pane in the Inspector and click on the Edit button for the Growables property. On the dialog which opens click on the column 1 and row 6 button and then on OK.

  • Post the changes you made and open the Designer again. You see that the last wx.TextCtrl is growing to fill the available space, this is due to the Flag 'wx.EXPAND', the Proportion '1' setting and the growable row/column settings. If you change the size of the designer window you see how your screen will react when a user changes its size.

  • It is always a good policy to change the names of your controls to something meaning full. This is as good a time as any to do this. I suggest to name the staticText controls as 'stFieldName' and the textCtrl as 'fieldName', but you should follow whater coding standards you use, or if you have none defined you could follow the one defined on this page http://wiki.wxpython.org/index.cgi/wxPython_Style_Guide . To do this select each control by clicking on it in the Designer window and change the properties in the Inspector. So, select the staticText1 control and in the Inspector Constr pane change the Label to 'First name' and the Name to 'stFirstName', then select the textCtrl1 and change Name to 'firstName' and Value to '' (blank).

  • Repeat the above for all fields.

  • Now your designer windows should look something like the following.




  • Oh, you mean it does not quit look like this :).

  • I manually changed the sizes of the controls, i.e. made them wider and also set some additional properties. On the address wx.TextCtrl I set the the property Style to 'wx.TE_MULTILINE' and on the notes control I set Style to 'wx.TE_RICH2 | wx.TE_MULTILINE'.

  • After these changes you should see a similar display when you run this little application – obviously one should add some validation and a way to store things, but this was not the point of this tutorial – maybe some other time.



Please note that the files generated during this example are also available in the directory "Examples\guide" under your Boa installation directory.
